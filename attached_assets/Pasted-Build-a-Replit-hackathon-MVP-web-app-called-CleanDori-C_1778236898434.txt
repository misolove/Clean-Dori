Build a Replit hackathon MVP web app called “클린돌이 CleanDori”.

CleanDori is an AI organizing coach.

Core idea:
A user uploads a messy room or workspace photo.
The app shows a cleaned-up motivational preview.
Then it gives one tiny, shame-free 3-minute organizing mission.

Important:
Build the mock version first.
Do not connect real APIs yet.
Do not add login.
Do not add database.
Do not add payments.
Do not overbuild.
The goal is a stable 3-minute hackathon demo.

Product message:
정리의 문제는 정보 부족이 아니라 시작 장벽입니다.
CleanDori는 사람을 진단하지 않습니다.
사진 속 공간 단서와 사용자 자기응답을 바탕으로 정리를 어렵게 만드는 실행 장벽을 추정하고,
정리 후 미리보기와 3분 미션으로 첫 행동을 돕습니다.

Use:
- React or Vite React
- Tailwind CSS if available
- Local state first
- Mock data first
- Korean-first UI
- Warm green / cream visual style
- Rounded cards
- Friendly, non-judgmental tone
- Desktop web layout first, mobile responsive if possible

Required screens / flow:

1. Landing Page
Title:
클린돌이 CleanDori

Subtitle:
사진 한 장으로 정리된 미래 모습을 보고, 3분 정리 미션을 시작하세요.

Key message:
정리의 문제는 정보 부족이 아니라 시작 장벽입니다.

CTA:
사진 올리고 시작하기

Secondary CTA:
Presentation Demo Mode

Small copy:
CleanDori는 당신의 공간을 판단하지 않습니다. 그냥 첫 3분을 도와드립니다.

2. Image Upload
- Let the user upload one image.
- Show uploaded image preview.
- Add “Use Demo Image” button.
- Add privacy warning:
  “민감한 문서, 신분증, 비밀번호, 개인정보가 보이는 사진은 올리지 마세요.”

3. User State Selector
Ask:
“지금 상태가 어때요?”

Options:
- 너무 막막해요 — 딱 하나만 알려줘요
- 에너지가 낮아요 — 3분만 가능해요
- 조금은 가능해요 — 5분 정도 가능해요
- 오늘은 해볼 만해요 — 10분 루틴 가능해요
- 버릴지 결정이 어려워요 — 보류 박스 방식이 필요해요

4. Before / Clean Preview
Show side-by-side:
- Left: 현재 모습
- Right: 정리 후 미리보기

Use fixed demo assets or mock placeholders first.

Add copy:
“완벽한 인테리어가 아니라, 지금 공간에서 어수선함만 줄인 동기부여용 미리보기입니다.”

Add optional button but keep it disabled or mock for now:
“AI로 새 애프터 생성하기”

5. Dual Coach Analysis
Show a short analysis section.

Title:
정리 전문가 + 비판단적 행동 코치

Show:
- 가능한 실행 장벽: 시각적 과부하, 시작점 상실, 결정 피로
- 부드러운 요약:
  “여러 물건이 섞여 있어서 어디서부터 시작할지 막막하게 느껴질 수 있어요. 전체를 치우지 않아도 괜찮아요. 지금은 한 구역만 회복해볼게요.”

Important:
Do not diagnose mental health conditions.
Do not mention ADHD, depression, OCD, hoarding disorder, anxiety, or any diagnosis.
Use “possible organizing barriers” only.

6. Mission Dashboard
For the default demo, use this mission:

User state:
너무 막막해요

Mission title:
바닥 통로 30cm 회복하기

Duration:
3분

Starting zone:
침대 앞 / 기타 옆 바닥 통로

Why this zone:
정리 후 미리보기에서 가장 먼저 달라져 보이는 구역입니다.

Steps:
1. 바닥에 있는 종이·빈 포장재처럼 명백한 것만 한곳에 모으기
2. 박스 밖으로 나온 작은 물건 3개만 박스 안에 넣기
3. 기타 주변 통로가 보이면 바로 멈추기

Avoid for now:
- 박스 전부 열기
- 추억 물건 판단하기
- 옷 정리 시작하기
- 방 전체를 완성하려고 하기

If-Then rule:
만약 바닥에 새 물건을 내려놓게 되면, 그때는 먼저 “보류 박스” 하나에만 넣는다.

7. Interactive Checklist
- User can check off each step.
- Show progress percentage.
- When complete, show:
  “좋아요. 통로 하나를 회복했어요. 완벽하지 않아도 괜찮아요. 오늘은 시작했고, 그게 핵심이에요.”

8. Presentation Demo Mode
Add a very stable Presentation Demo Mode button.

When clicked:
- Use demo before image or placeholder.
- Use demo after image or mock clean preview.
- Set user state to “너무 막막해요.”
- Show the Dual Coach Analysis.
- Show the default 3-minute mission.
- Checklist must work.
- Completion message must work.

Important architecture:
Create mock-first functions:
- analyzeRoomImage(image, userState)
- generateCleaningMission(analysis, userState)
- generateCleanPreviewPrompt(analysis, userState)
- generateCleanPreviewImage(image, prompt)
- compareAfterPhoto(beforeImage, previewImage, afterImage)

For now, all functions return mock data.

Future API hooks:
- Claude API will later be used for room analysis and 3-minute mission JSON.
- Gemini API will later be used for cleaned-up preview image generation.
- Do not implement these APIs yet in the first pass.
- But leave clean function boundaries so they can be added later.

Safety and tone rules:
- Never shame the user.
- Never say the room is dirty.
- Never say the user is lazy.
- Never tell the user to clean the whole room.
- Never diagnose the user.
- If safety risk is visible in the future, prioritize safety guidance.

Design requirements:
- Desktop browser version should be intuitive.
- Prefer a 3-column layout:
  Left: upload and user state
  Center: before / clean preview
  Right: analysis and mission checklist
- Use warm, calm colors.
- Make it polished enough for a hackathon demo.
- Keep text short and clear.

After building:
- Run the app.
- Fix runtime errors.
- Make sure Presentation Demo Mode works.
